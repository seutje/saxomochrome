Tone.FatOscillator

new Tone.FatOscillator (frequency, type, modulationType)
frequency {Frequency}

The starting frequency of the oscillator.

type {String}

The type of the carrier oscillator.

modulationType {String}

The type of the modulator oscillator.

code
//a sine oscillator frequency-modulated by a square wave
var fmOsc = new Tone.FatOscillator("Ab3", "sine", "square").toMaster().start();
default values
frequency:440
detune:0
phase:0
spread:20
count:3
type:"sawtooth"
members
.frequency ~ {Frequency}

The oscillator's frequency

.detune ~ {Cents}

The detune control signal.

.type {string}

The type of the carrier oscillator

.spread {Cents}

The detune spread between the oscillators. If "count" is set to 3 oscillators and the "spread" is set to 40, the three oscillators would be detuned like this: [-20, 0, 20] for a total detune spread of 40 cents.

.count {Number}

The number of detuned oscillators

.phase {Number}

The phase of the oscillator in degrees.

.partials {Array}

The partials of the carrier waveform. A partial represents the amplitude at a harmonic. The first harmonic is the fundamental frequency, the second is the octave and so on following the harmonic series. Setting this value will automatically set the type to "custom". The value is an empty array when the type is not "custom".

code
osc.partials = [1, 0.2, 0.01];
.volume ~ {Decibels}

The volume of the output in decibels.

code
source.volume.value = -6;
.state {Tone.State} READ ONLY

Returns the playback state of the source, either "started" or "stopped".

.mute {boolean}

Mute the output.

code
//mute the output
source.mute = true;

.syncFrequency ()
returns {Tone.Oscillator}

this

Sync the signal to the Transport's bpm. Any changes to the transports bpm, will also affect the oscillators frequency.

code
Tone.Transport.bpm.value = 120;
osc.frequency.value = 440;
//the ration between the bpm and the frequency will be maintained
osc.syncFrequency();
Tone.Transport.bpm.value = 240; 
// the frequency of the oscillator is doubled to 880
