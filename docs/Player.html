Player → Tone.Source

Tone.Player is an audio file player with start, loop, and stop functions.

new Tone.Player (url[, onload])
url {string|AudioBuffer}

Either the AudioBuffer or the url from which to load the AudioBuffer

onload {function}

The function to invoke when the buffer is loaded. Recommended to use Tone.Buffer.onload instead.

code
var player = new Tone.Player("./path/to/sample.mp3").toMaster();
//play as soon as the buffer is loaded
player.autostart = true;
default values
onload:Tone.noOp
playbackRate:1
loop:false
autostart:false
loopStart:0
loopEnd:0
retrigger:false
reverse:false
members
.autostart {boolean}

If the file should play as soon as the buffer is loaded.

code
//will play as soon as it's loaded
var player = new Tone.Player({
	"url" : "./path/to/sample.mp3",
	"autostart" : true,
}).toMaster();
.retrigger {boolean}

Enabling retrigger will allow a player to be restarted before the the previous 'start' is done playing. Otherwise, successive calls to Tone.Player.start will only start the sample if it had played all the way through.

.loopStart {Time}

If loop is true, the loop will start at this position.

.loopEnd {Time}

If loop is true, the loop will end at this position.

.buffer {Tone.Buffer}

The audio buffer belonging to the player.

.loop {boolean}

If the buffer should loop once it's over.

.playbackRate {number}

The playback speed. 1 is normal speed. This is not a signal because Safari and iOS currently don't support playbackRate as a signal.

.reverse {boolean}

The direction the buffer should play in

.volume ~ {Decibels}

The volume of the output in decibels.

code
source.volume.value = -6;
.state {Tone.State} READ ONLY

Returns the playback state of the source, either "started" or "stopped".

.mute {boolean}

Mute the output.

code
//mute the output
source.mute = true;
methods
.load (url[, callback])
url {string}
The url of the buffer to load. Filetype support depends on the browser.
callback {function}
The function to invoke once the sample is loaded.
returns {Tone.Player}

this

Load the audio file as an audio buffer. Decodes the audio asynchronously and invokes the callback once the audio buffer loads. Note: this does not need to be called if a url was passed in to the constructor. Only use this if you want to manually load a new url.

.start ([startTime][, offset][, duration])
startTime = now {Time}
When the player should start.
offset = 0 {Time}
The offset from the beginning of the sample to start at.
duration {Time}
How long the sample should play. If no duration is given, it will default to the full length of the sample (minus any offset)
returns {Tone.Player}

this

Play the buffer at the given startTime. Optionally add an offset and/or duration which will play the buffer from a position within the buffer for the given duration.

.setLoopPoints (loopStart, loopEnd)
loopStart {Time}
The loop end time
loopEnd {Time}
The loop end time
returns {Tone.Player}

this

Set the loop start and end. Will only loop if loop is set to true.

code
//loop 0.1 seconds of the file. 
player.setLoopPoints(0.2, 0.3);
player.loop = true;
.dispose ()
returns {Tone.Player}

this

Dispose and disconnect.

.stop ([time])
time = now {Time}
When the source should be stopped.
returns {Tone.Source}

this

Stop the source at the specified time. If no time is given, stop the source now.

code
source.stop(); // stops the source immediately
.sync ([delay])
delay = 0 {Time}
Delay time before starting the source after the Transport has started.
returns {Tone.Source}

this

Sync the source to the Transport so that when the transport is started, this source is started and when the transport is stopped or paused, so is the source.

code
//sync the source to start 1 measure after the transport starts
source.sync("1m");
//start the transport. the source will start 1 measure later. 
Tone.Transport.start();
.unsync ()
returns {Tone.Source}

this

Unsync the source to the Transport. See Tone.Source.sync
